### 2. SERVICE DESCRIPTION AND FUNCTIONALITY

The Service is a social networking and event discovery platform designed to connect users based on shared interests and local events. Core functionalities include:
*   **Event Discovery:** Users can browse, view details, and discover local events.
*   **Event Management:** Users can create, manage, and view details of events, which are subject to administrative review and approval.
*   **Social Interaction:** Users can join events, bookmark events, and communicate via private messaging (Chat).

### 4. DATA COLLECTION AND PRIVACY DISCLOSURES

**A. Data Collected:**
The Service collects various types of personal and usage data, including but not limited to:
*   **Identity Data:** Full name, email address, profile picture, date of birth, gender, and user-provided bio information.
*   **Location Data:** Precise and approximate location data (via `expo-location`) is collected when the user grants permission, primarily for displaying nearby events.
*   **Communication Data:** Content of messages sent through the chat feature.
*   **Activity Data:** Records of events attended, events bookmarked, and user interactions within the app.
*   **Media Data:** Photos and images uploaded by the user (e.g., profile pictures, event photos).

**B. Permissions:**
The Service requires and utilizes the following device permissions:
*   **Location Services:** Required for geo-fencing and displaying local events.
*   **Photo Library Access:** Required for uploading profile pictures and event media.

**C. Data Storage:**
Data is stored using a combination of technologies:
*   **Local Storage:** AsyncStorage is used for temporary client-side data.
*   **Secure Storage:** Expo SecureStore is used for highly sensitive data, such as authentication tokens.
*   **Database:** A PostgreSQL database is used to manage core data structures, including `app_user`, `event`, `notification`, `conversation`, and `message` tables.
